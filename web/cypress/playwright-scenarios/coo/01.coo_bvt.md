## Environment Variables

> All values are sourced from `cypress/export-env.sh`. Run `source cypress/export-env.sh` before executing this scenario.

| Variable | `export-env.sh` key | How to extract |
|----------|---------------------|----------------|
| `BASE_URL` | `CYPRESS_BASE_URL` | Use directly |
| `LOGIN_IDP` | `CYPRESS_LOGIN_IDP` | Use directly (e.g. `kube:admin` or `my_htpasswd_provider`) |
| `LOGIN_USERNAME` | `CYPRESS_LOGIN_USERS` | First entry before `:` — `echo $CYPRESS_LOGIN_USERS \| cut -d',' -f1 \| cut -d':' -f1` |
| `LOGIN_PASSWORD` | `CYPRESS_LOGIN_USERS` | First entry after `:` — `echo $CYPRESS_LOGIN_USERS \| cut -d',' -f1 \| cut -d':' -f2` |
| `KUBECONFIG_PATH` | `CYPRESS_KUBECONFIG_PATH` | Use directly |
| `COO_NAMESPACE` | `CYPRESS_COO_NAMESPACE` | Use directly (default: `openshift-cluster-observability-operator`) |
| `SKIP_ALL_INSTALL` | `CYPRESS_SKIP_ALL_INSTALL` | Use directly (`true` or `false`) |

## Prerequisites (CLI)

> Skip this section if `CYPRESS_SKIP_ALL_INSTALL=true`.

### P1. Clean up previous COO installation

```shell
oc delete uiplugin monitoring --ignore-not-found --kubeconfig {KUBECONFIG_PATH}
oc delete csv --all -n {COO_NAMESPACE} --ignore-not-found --wait=false --kubeconfig {KUBECONFIG_PATH}
oc delete subscription --all -n {COO_NAMESPACE} --ignore-not-found --wait=false --kubeconfig {KUBECONFIG_PATH}
```

### P2. Revert monitoring plugin image

```shell
oc -n openshift-monitoring patch consoles.operator.openshift.io cluster --type=json -p='[{"op":"remove","path":"/spec/plugins"}]' --kubeconfig {KUBECONFIG_PATH}
```

### P3. Install COO operator

> Installation method depends on environment configuration.
> See `cypress/README.md` and `cypress/support/commands/coo-install-commands.ts` for details.

### P4. Wait for COO operator ready

```shell
oc get pods -n {COO_NAMESPACE} --kubeconfig {KUBECONFIG_PATH} | grep cluster-observability-operator
```

Repeat until pod status is `Running`.

### P5. Set up monitoring UIPlugin

```shell
oc apply -f cypress/fixtures/coo/uiplugin-monitoring.yaml --kubeconfig {KUBECONFIG_PATH}
```

### P6. Set up dashboards and plugins

```shell
oc apply -f cypress/fixtures/coo/perses-dashboards.yaml --kubeconfig {KUBECONFIG_PATH}
```

### P7. Set up troubleshooting panel

```shell
oc apply -f cypress/fixtures/coo/troubleshooting-panel.yaml --kubeconfig {KUBECONFIG_PATH}
```

### P8. Set up monitoring plugin image

```shell
oc -n openshift-monitoring patch consoles.operator.openshift.io cluster --type=merge -p '{"spec":{"plugins":["monitoring-console-plugin"]}}' --kubeconfig {KUBECONFIG_PATH}
```

### P9. Remove cluster-admin role

```shell
oc adm policy remove-cluster-role-from-user cluster-admin {LOGIN_USERNAME} --kubeconfig {KUBECONFIG_PATH}
```

## Setup (Browser)

### S1. Navigate to console

> The console URL comes from `CYPRESS_BASE_URL` in `cypress/export-env.sh`.
> If a certificate error appears after navigation, follow the **Certificate acceptance** rule from the [README](../README.md): click "Advanced" → click "Proceed to ... (unsafe)". This applies here and to every subsequent navigation in this scenario.

| Step | Action | Target | Expected Result |
|------|--------|--------|-----------------|
| S1.1 | navigate | `$CYPRESS_BASE_URL` | Page loads (accept certificate if prompted) |
| S1.2 | wait | 5 seconds | OAuth page loads |

### S2. Login via OAuth

> Credentials come from `cypress/export-env.sh`:
> - **IDP**: `$CYPRESS_LOGIN_IDP` (e.g. `kube:admin` or `my_htpasswd_provider`)
> - **Username**: first entry in `$CYPRESS_LOGIN_USERS` before the `:`
> - **Password**: first entry in `$CYPRESS_LOGIN_USERS` after the `:`
>
> Extract with:
> ```shell
> USERNAME=$(echo $CYPRESS_LOGIN_USERS | cut -d',' -f1 | cut -d':' -f1)
> PASSWORD=$(echo $CYPRESS_LOGIN_USERS | cut -d',' -f1 | cut -d':' -f2)
> ```

| Step | Action | Target | Expected Result |
|------|--------|--------|-----------------|
| S2.1 | snapshot | | Capture OAuth IDP selection page |
| S2.2 | click | Link matching `$CYPRESS_LOGIN_IDP` text (e.g. "kube:admin" or "my_htpasswd_provider") | Login form appears (accept certificate if prompted) |
| S2.3 | snapshot | | Capture login form |
| S2.4 | fill | Username field (`#inputUsername` or textbox "Username") with extracted USERNAME | Username entered |
| S2.5 | fill | Password field (`#inputPassword` or textbox "Password") with extracted PASSWORD | Password entered |
| S2.6 | click | "Log in" button (`button[type=submit]` or button containing "Log in") | Authentication proceeds (accept certificate if prompted on redirect) |
| S2.7 | wait | 30 seconds | Console loads after redirect |
