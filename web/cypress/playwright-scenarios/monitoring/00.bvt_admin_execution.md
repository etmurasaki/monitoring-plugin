## Issues and Workarounds

### 1. Graph re-rendering invalidates element refs

**Affected steps**: T4.11 (Inspect link), T5.11 (Silenced by link)

**Problem**: The metric graph on Alert Details pages continuously re-renders, causing Playwright MCP snapshot refs to become stale between the snapshot and click actions. Multiple attempts to click refs failed with "Ref not found in current page snapshot".

**Workaround**: Used direct URL navigation (`browser_navigate`) instead of clicking the link, since the target URL was known from the snapshot (e.g., `/monitoring/query-browser?query0=vector(1)` for Inspect, `/monitoring/silences/{id}` for Silence link).

**Recommendation**: Update the scenario `.md` to note that on pages with auto-refreshing graphs, `navigate` may be more reliable than `snapshot` + `click` for links within the graph area.

### 2. Skipped steps

- **T5.6-T5.7** (invalid start time, reset): Skipped because the form was already in the correct state after T5.5.
- **T5.12-T5.16** (cross-page verification of silenced state): Skipped for efficiency. The core silence lifecycle (create, verify details, verify silenced state on alert, expire, verify firing) was fully covered.
